## 框架目标

提供一个系统性的 **AI Benchmark 质量评估框架**，通过 46 条最佳实践评估 benchmark 全生命周期的质量。

## Benchmark 生命周期模型

BetterBench 将 benchmark 分为以下生命周期阶段，每个阶段有对应的评估标准：

1. **概念定义** → 测试什么能力
2. **任务设计** → 如何将能力转化为任务
3. **数据构建** → 数据来源和质量
4. **评估实现** → 代码和工具
5. **结果报告** → 指标和可解释性
6. **维护更新** → 长期可用性

## 46 条最佳实践（分类整理）

### 一、概念与定义 (Concept & Definition)

| # | 标准 | 评估方式 |
|---|------|----------|
| 1 | 明确定义测试的能力/特征 | 是否显式提及并定义 |
| 2 | 描述能力如何转化为 benchmark 任务 | 任务与能力的对应关系 |
| 3 | 说明测试概念的现实价值 | 为什么了解此能力有用 |
| 4 | 描述使用场景和用户画像 | 文化/地理背景、交互类型 |
| 5 | 领域专家参与 | 共同作者或设计顾问 |
| 6 | 整合领域文献 | 文献如何影响设计 |

### 二、指标与基线 (Metrics & Baselines)

| # | 标准 | 评估方式 |
|---|------|----------|
| 7 | 说明分数的正确/错误解读方式 | 防止误用 |
| 8 | 选择有理由的性能指标 | 非标准指标需解释 |
| 9 | 提供指标的上下界 | floor/ceiling |
| 10 | 包含人类表现水平 | 如何测量、专家/普通人 |
| 11 | 包含随机表现水平 | 随机基线 |

### 三、鲁棒性 (Robustness)

| # | 标准 | 评估方式 |
|---|------|----------|
| 12 | 处理输入敏感性 | 多种语义等价输入 |
| 13 | 验证自动评估 | 自动评估方法的验证 |

### 四、可复现性 (Reproducibility)

| # | 标准 | 评估方式 |
|---|------|----------|
| 14 | 与相关 benchmark 的差异说明 | 填补什么空白 |
| 15 | 评估代码公开 | GitHub 仓库 |
| 16 | 结果复现脚本 | 输入+输出+代码+超参 |
| 17 | 评估数据/prompt/环境可访问 | 不必公开但需可评估 |
| 18 | 支持 API 模型评估 | 至少两种 API |
| 19 | 支持本地模型评估 | 无需 API |

### 五、防污染 (Contamination Prevention)

| # | 标准 | 评估方式 |
|---|------|----------|
| 20 | 全局唯一标识符或加密 | GUID/canary string |
| 21 | 训练集检测任务 | 检测模型是否在测试集上训练 |

### 六、安全与伦理 (Safety & Ethics)

| # | 标准 | 评估方式 |
|---|------|----------|
| 22 | 敏感/有害内容警告 | 评估任务和预期输出 |
| 23 | 发布要求说明 | 使用条款和限制 |

### 七、代码质量 (Code Quality)

| # | 标准 | 评估方式 |
|---|------|----------|
| 24 | 构建状态指示 | CI/CD |
| 25 | 依赖文件 | requirements.txt |
| 26 | 快速入门指南 | 步骤说明 |
| 27 | 代码注释 | 目的、输入、输出 |
| 28 | 代码文档 | 目录结构、函数说明 |

### 八、任务设计 (Task Design)

| # | 标准 | 评估方式 |
|---|------|----------|
| 29 | 任务分类与理由文档 | 为什么选这些任务 |
| 30 | 规范性属性假设 | 文化背景差异 |
| 31 | 标注者/数据源的人口统计 | 多样性 |

### 九、数据质量 (Data Quality)

更多标准涵盖数据收集、标注质量、统计显著性等。

## 评分规则

每条标准 0-15 分（4 档）：
- **0 分**：完全没有
- **5 分**：提及但未实现
- **10 分**：部分实现
- **15 分**：完全实现

## 核心发现

1. **大多数 benchmark 不报告统计显著性**
2. **结果难以复现**
3. **常用 benchmark 之间质量差异巨大**
4. **实现不一致和 prompt 工程复杂度被忽视**

## 实际应用价值

- 可作为评估任何 benchmark 质量的 checklist
- 帮助 benchmark 开发者识别改进方向
- 帮助用户选择高质量的 benchmark
